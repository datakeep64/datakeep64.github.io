ML Pipeline Automation System

A reproducible, production-style ML workflow for news topic classification — TF-IDF feature extraction, automated multi-model selection by 5-fold CV, evaluation with threshold gating, and a FastAPI text-classification endpoint — containerised with Docker and automated via GitHub Actions CI/CD.

Python Docker GitHub Actions Scikit-learn TF-IDF FastAPI Pytest 20 Newsgroups

Pipeline Stages

A linear workflow designed for reproducibility — every run is deterministic, containerised, and logged. Each stage writes named artifacts so any step can be re-run independently.

01

Data Preprocessing

Fetches 20 Newsgroups articles for 4 topic categories (Space, Sports, Politics, Technology). Strips email headers, footers, and quoted replies so the model learns from article body text rather than metadata. Fits a TF-IDF vectoriser (2,000 terms, sublinear TF) on training articles and saves the vectoriser, label map, and vocabulary as versioned artifacts.

02

Multi-Model Selection

Three candidates are evaluated by 5-fold cross-validated macro-OVR AUC: LogisticRegression (strong linear TF-IDF baseline), RandomForest (non-linear ensemble), and SGDClassifier (online SGD — O(n) training time, designed for when the corpus grows to millions of articles). The best model is auto-selected and saved without manual intervention.

03

Evaluation with Threshold Gate

Computes macro-OVR AUC, macro F1, per-class precision/recall, and confusion matrix on the held-out test split. Runs permutation importance across all 2,000 TF-IDF features — model-agnostic, so it works equally for RF, LR, and SGD. Fails the CI run automatically if AUC drops below 0.92.

04

FastAPI Inference Endpoint

Accepts raw article text, vectorises via the saved TF-IDF vocabulary, and returns the predicted topic with per-class probability scores. A /explain endpoint exposes the model comparison and top-10 permutation-important terms for auditability.

05

CI/CD via GitHub Actions

On every push: installs dependencies, runs the 9-test Pytest suite, executes the full preprocess → train → evaluate pipeline, and builds the Docker image. Acts as a regression gate — a drop in model quality fails the build before it can reach production.
